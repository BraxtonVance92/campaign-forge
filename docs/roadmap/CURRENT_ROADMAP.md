## Contest score strategy

### Real-world utility

Demonstrate that the creator fingerprint changes the generated brief/output and saves a creator from starting over. Show the initial example, extracted patterns, editable correction, and resulting asset.

### Production readiness

Show consent, file validation, real states, persisted restore, failure/retry, idempotency, a tested deployment, setup documentation, and judge-friendly access.

### B2 storage and orchestration

Make B2 visible as the durable project memory: source, fingerprint versions, intermediates, final assets, and manifest. Demonstrate refresh/restore and lineage, not just a hidden upload.

### Genblaze use

Use Genblaze for at least a multi-step creative brief/reference -> image -> optional video workflow with real events, provider adapter, B2 sink, canonical manifest, and lineage. Do not make a direct GMI call where the adapter supports the operation.

## Cut line

Required: Blocks 0-3, judge-ready essentials from Block 5, README/video/submission, live deployment.

Cut first: voice, avatar, social publishing, analytics dashboards, teams, billing, multiple model fan-out, fancy editing, and more than one output format.
